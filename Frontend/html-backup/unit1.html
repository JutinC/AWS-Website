Unit 1 – Cloud Concepts Overview

Cloud Computing • AWS Basics • Global Infrastructure

Unit 1 Overview

AWS Unit 1 introduces the fundamental concepts of cloud computing and explains how cloud technology is transforming the way organizations build and manage IT infrastructure. Instead of relying on physical servers and on-premises data centers, cloud computing allows businesses to access computing resources over the internet.

Cloud computing provides flexible, scalable, and cost-efficient solutions for storing data, running applications, and managing infrastructure. This unit focuses on understanding what the cloud is, why it is beneficial, and how AWS plays a role in delivering cloud services globally.

Introduction to Cloud Computing

Cloud computing is the delivery of computing services—such as servers, storage, databases, networking, and software—over the internet. Instead of owning and maintaining physical hardware, users can access these resources on demand from cloud providers like AWS.

This model allows organizations to quickly scale resources up or down depending on their needs, removing the need for large upfront investments in hardware and reducing the complexity of managing infrastructure.

Service Models:
  • IaaS (Infrastructure as a Service) – Virtualized computing resources such as Amazon EC2.
  • PaaS (Platform as a Service) – Managed platforms for building applications.
  • SaaS (Software as a Service) – Fully managed applications accessed over the internet.

Advantages of the Cloud

Cloud computing offers several key advantages compared to traditional on-premises systems.

Key Benefits:
  • Scalability – Instantly increase or decrease resources based on demand.
  • Cost Efficiency – Pay only for what you use instead of buying hardware upfront.
  • Availability & Reliability – Built-in redundancy across multiple data centers.
  • Global Reach – Deploy applications closer to users around the world to reduce latency.

Introduction to AWS

Amazon Web Services (AWS) is one of the leading cloud service providers in the world, offering a wide range of services including compute, storage, databases, networking, analytics, and machine learning.

AWS operates on a global infrastructure made up of Regions and Availability Zones. Regions are geographic areas, while Availability Zones are isolated data centers within a region designed for fault tolerance and high availability.

AWS uses a pay-as-you-go pricing model, meaning users are charged only for the resources they consume, making it accessible to both startups and large enterprises.

AWS Global Infrastructure

AWS infrastructure is designed for high availability and reliability. Each Region contains multiple Availability Zones that are physically separated but connected with high-speed networking.

This design allows applications to be distributed across multiple locations so that if one data center fails, others can continue operating. AWS also uses edge locations to deliver content quickly to users worldwide, improving performance for latency-sensitive applications.

Moving to the AWS Cloud

Migrating to the AWS cloud involves transferring applications, data, and workloads from on-premises systems to AWS infrastructure. This allows organizations to gain scalability, flexibility, and cost savings.

The “6 R’s” of Migration:
  • Rehosting (lift and shift)
  • Replatforming
  • Repurchasing
  • Refactoring
  • Retiring
  • Retaining

Benefits of Cloud Migration

Moving to the cloud provides greater flexibility, enabling faster deployment of applications and quicker responses to changing business needs. Disaster recovery improves because data can be replicated across multiple locations.

By spending less time managing physical infrastructure, organizations can focus more on innovation—developing new features, improving user experience, and experimenting with new ideas.

Shared Responsibility Model

AWS uses a shared responsibility model to define how security duties are divided between AWS and the customer.

Responsibilities:
  • AWS is responsible for security of the cloud (infrastructure, hardware, and facilities).
  • Customers are responsible for security in the cloud (applications, data, configurations, and access).

Unit 1 Summary

AWS Unit 1 introduces the core concepts of cloud computing and explains how cloud services provide scalable, reliable, and cost-efficient solutions for modern applications.

Cloud computing removes the need for physical infrastructure and allows organizations to access resources on demand. AWS provides a global platform for delivering these services, supported by Regions, Availability Zones, and edge locations.

Understanding these concepts is essential for building and managing cloud-based systems and sets the foundation for all future AWS topics.
