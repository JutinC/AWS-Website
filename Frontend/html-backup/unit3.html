Unit 3 – AWS Global Infrastructure

Regions • Availability Zones • Edge Locations

Unit 3 Overview

AWS Unit 3 focuses on the global infrastructure that allows AWS to deliver cloud services reliably and efficiently around the world. Cloud computing depends heavily on physical data centers, networking systems, and geographic distribution to ensure high availability and performance.

AWS has one of the largest global infrastructures, consisting of Regions, Availability Zones, and Edge Locations. This structure allows applications to run closer to users, reduce latency, and maintain availability even when failures occur.

AWS Global Infrastructure

AWS global infrastructure refers to the physical and virtual systems that support AWS services worldwide, including data centers, networking, and locations that host cloud resources.

It is designed to provide high availability, support fault tolerance, deliver low latency, and enable global scalability by distributing resources across multiple geographic areas.

AWS Regions

An AWS Region is a geographic area that contains multiple data centers. Each Region is isolated from others to improve fault tolerance and security.

Choosing a Region:
  • Proximity to users to reduce latency.
  • Compliance and data residency requirements.
  • Service availability and feature support.
  • Cost differences between Regions.

Using multiple Regions can improve disaster recovery and ensure applications remain available during large-scale outages.

Availability Zones (AZs)

Availability Zones are isolated data centers within a Region. They are physically separated but connected with high-speed, low-latency networking.

AZs are designed so that failures in one zone do not affect others. By deploying resources across multiple AZs, organizations can build highly available systems that continue running even if one data center fails.

Edge Locations

Edge locations are smaller data centers located closer to end users. They are primarily used for content delivery and caching through services like Amazon CloudFront.

Delivering content from edge locations reduces latency, improves user experience, and enables fast global content delivery instead of sending data from a single central Region.

High Availability & Fault Tolerance

AWS infrastructure is designed to support both high availability and fault tolerance.

Key Concepts:
  • High Availability – Systems are designed to remain operational with minimal downtime by using multiple AZs and Regions.
  • Fault Tolerance – Systems can continue operating even when components fail through redundancy and automatic failover.

Using multiple AZs and Regions, along with redundant networking and storage, helps applications stay online even during failures.

AWS Services Overview

AWS offers many services that run on top of its global infrastructure. These services are organized into categories based on their function.

Major Service Categories:
  • Compute – Run applications (e.g., EC2, Lambda).
  • Storage – Store and manage data (e.g., S3, EBS).
  • Database – Manage structured and unstructured data (e.g., RDS, DynamoDB).
  • Networking – Connect resources and manage traffic (e.g., VPC, Route 53).
  • Security – Protect systems and data (e.g., IAM, KMS).

Unit 3 Summary

AWS Unit 3 explains how AWS uses a global network of Regions, Availability Zones, and Edge Locations to deliver cloud services efficiently.

Regions provide geographic separation, AZs enable high availability, and edge locations improve performance for users worldwide. Understanding this infrastructure is essential for designing systems that can handle failures, scale globally, and deliver consistent performance.
