Unit 8 – Database Services

Amazon RDS • Amazon Aurora • Amazon DynamoDB • Amazon Redshift

Unit 8 Overview

AWS Unit 8 focuses on database services, explaining how structured data is stored, queried, and managed at scale. Databases are essential for applications that require fast access to organized data, such as user accounts, transactions, analytics, and reporting systems.

This unit explores different types of databases including relational databases, NoSQL databases, data warehouses, and cloud-native relational databases. Each service is designed for specific workloads, performance needs, and scalability requirements. By the end of this unit, you should be able to choose the most appropriate database service for a given workload.

Database Basics

Databases store structured or semi-structured data in a way that allows it to be efficiently queried, updated, and managed. Unlike simple file storage, databases support indexing, relationships, and optimized queries. AWS provides multiple database services because applications have different needs.

Some applications require strict consistency and immediate updates, while others need massive scalability and flexible data models. Understanding the difference between relational and NoSQL databases is critical for selecting the right database service for your workload.

Amazon RDS (Relational Database Service)

Amazon RDS is a managed relational database service that supports popular database engines such as MySQL, PostgreSQL, MariaDB, Oracle, and SQL Server. It is designed for easy setup, operation, and scaling. It automates tasks such as hardware provisioning, database setup, patching, and backups. It is ideal for familiar SQL-based databases.

Amazon Aurora

Aurora is a fully managed relational database built by AWS and compatible with MySQL and PostgreSQL. It combines the familiarity of traditional databases with the performance and reliability that cloud-native services bring. Aurora offers high availability, automatic scaling, and improved durability compared to traditional RDS engines.

RDS - How It Works

With Amazon RDS, AWS manages the underlying infrastructure, operating system, and database software. Users interact with the databases through standard SQL queries, just as they would with a traditional relational database.

RDS supports Multi-AZ deployment for high availability and automatic failover. AWS handles backups, software patching, and monitoring, allowing developers to focus on application development rather than database management.

Aurora - How It Differs from RDS

  • Unlike traditional RDS engines, Aurora separates compute and storage. This allows storage to scale transparently without downtime and improves performance.
  • Proactive features include automatic patching, scaling, and improved durability with replication across multiple Availability Zones.

RDS - Use Cases & Benefits

Common use cases include web applications, e-commerce platforms, transaction processing, and applications requiring reliable relational database performance.

Benefits include ease of use, automated management, scalability, and reliability. RDS simplifies database administration while maintaining compatibility with existing applications.

Aurora - Use Cases

  • High-traffic web applications
  • SaaS platforms
  • Applications requiring high availability and managed scalability
  • Mobile and web applications
  • Gaming leaderboards
  • IoT applications and session management

Amazon Redshift

Amazon Redshift is a fully managed data warehouse service designed for large-scale data analytics. Unlike transactional databases, Redshift is optimized for analyzing massive datasets through parallel processing. It can handle petabytes of data and supports complex analytical queries for business intelligence and reporting.

Amazon DynamoDB

Amazon DynamoDB is a fully managed NoSQL database service designed for applications that need consistent, single-digit millisecond latency at any scale. Unlike relational databases, DynamoDB provides flexible data models and automatic scaling without the need for server management.

Database Comparison

  • Amazon RDS: Managed relational database service, suitable for different application requirements. By understanding the strengths and interactions of RDS, Aurora, DynamoDB, and Redshift, organizations can design data architectures that are efficient, scalable, and cost-effective.
  • Amazon Aurora: Cloud-native relational database with high performance, automatic scaling, and improved durability compared to traditional RDS.
  • Amazon DynamoDB: Serverless NoSQL database with extreme scalability and single-digit millisecond latency.
  • Amazon Redshift: Data warehouse optimized for analytics and reporting, designed for large-scale data processing.

Choosing the right database service depends on data structure, access patterns, scalability needs, and performance requirements.
