Unit 5 – Networking & Content Delivery

Amazon VPC • VPC Wizard • VPC Networking • VPC Security • Route 53 • CloudFront

Unit 5 Overview

AWS Unit 5 focuses on cloud networking, how virtual networks are built using Amazon VPC, how traffic flows through subnets and route tables; and how AWS delivers content globally using Route 53 and CloudFront. The goal of this unit is to understand how AWS resources communicate, how to control access using networking rules, and how to improve performance using content delivery services.

Amazon VPC (Virtual Private Cloud)

Amazon VPC is a logically isolated virtual network in AWS where you can launch resources like EC2 instances. It is similar to having your own private data center in the cloud.

You control:
  • IP address ranges (CIDR blocks)
  • Subnets
  • Route tables
  • Internet gateways
  • Security rules
Note: Every AWS account comes with a default VPC, but you can create custom VPCs for better security and organization.

Subnets

A subnet is a smaller network inside a VPC. Each subnet exists in only one Availability Zone.

Types of subnets:
  • Public subnet - has a route to an Internet Gateway
  • Private subnet - no direct internet access
Note: Subnets are used to organize workloads and improve security.

Route Tables

Route tables control how traffic is directed within your VPC and to external networks.

Common routes:
  • Local - Internal VPC traffic
  • 0.0.0.0/0 - internet traffic
Note: If a route table is misconfigured, resources may not be reachable.

CIDR & IP Addressing

CIDR (Classless Inter-Domain Routing) defines the IP address range for your VPC and subnets.

Example: 10.0.0.0/16

The /16 means there are 65,536 IP addresses.

Note: CIDR blocks cannot be changed after creation.

Security Groups

Security Groups are virtual firewalls for EC2 instances that control inbound and outbound traffic.

Features:
  • Stateful - return traffic is automatically allowed
  • Applied at the instance level
  • Can have multiple security groups per instance
Note: Security Groups are the primary way to control network access to your instances.

Route 53

Route 53 is AWS's DNS (Domain Name System) service.

Features:
  • Domain registration
  • DNS routing
  • Health checks
Note: Routing policies include simple, weighted, latency, Failover, and geolocation.

Unit 5 Summary

AWS Unit 5 focuses on networking and content delivery.

Key Points:
  • Amazon VPC provides isolated networks
  • Subnets divide networks
  • Route tables control traffic
  • Security Groups and NACLs provide security
  • Route 53 handles DNS and domain management
  • CloudFront delivers content globally
