Unit 6 – Compute Services

Amazon EC2 • Cost Optimization • Containers • Lambda • Elastic Beanstalk

Unit 6 Overview

AWS Unit 6 concentrates on compute services, which define how applications run in the cloud. Compute services deliver processing power, memory, and execution environments for applications. This unit delves into different compute models, including virtual machines, containers, managed platforms, and serverless computing. It explains how AWS enables developers to balance control, cost, and operational responsibility.

Amazon EC2 (Elastic Compute Cloud)

Amazon EC2 provides resizable virtual servers in the cloud. Users can launch virtual machines that scale up or down based on demand, eliminating the need to purchase and maintain physical servers.

Service Model:

EC2 is classified as Infrastructure as a Service (IaaS).

Responsibility Split:
  • AWS manages the underlying hardware
  • Customers are responsible for the operating system, applications, and configurations

EC2 - How It Works

When an EC2 instance is launched, AWS provisions a virtual machine with the specified configuration.

Process:
  • Select an Amazon Machine Image (AMI)
  • Choose an instance type
  • Configure security groups and networking
  • Launch the instance

EC2 - Instance Types

EC2 offers multiple instance families, each optimized for different workloads.

Instance Families:
  • General purpose instances: Offer balanced resources
  • Compute-optimized instances: Focus on performance
  • Memory-optimized instances: Designed for large in-memory databases
  • Storage-optimized instances: Provide fast disk access

EC2 - Benefits & Use Cases

EC2 is ideal for applications that require full control over the server.

Benefits:
  • Supports custom software stacks
  • Compatible with legacy applications
  • Handles specialized workloads
Common Use Cases:
  • Web servers
  • Application servers
  • Development environments

EC2 Cost Optimization

AWS provides multiple pricing models to help reduce compute costs. Cost optimization focuses on matching options to specific needs.

Pricing Models:
  • On-Demand: Pay for compute capacity by the hour or second
  • Reserved Instances: Significant discount for 1-3 year commitments
  • Spot Instances: Use spare capacity at up to 90% discount
  • Savings Plans: Flexible pricing model with lower costs

Amazon ECS (Elastic Container Service)

Amazon ECS is a fully managed container orchestration platform that allows developers to run and manage containers.

Features:
  • Fully managed service
  • Supports Docker containers
  • Automatic scaling
  • Integrated with other AWS services

Amazon EKS (Elastic Kubernetes Service)

Amazon EKS runs Kubernetes on AWS, making it easy to deploy, manage, and scale containerized applications.

Ideal For:
  • Teams already using Kubernetes
  • Requiring portability across environments
  • Need for Kubernetes-native features

AWS Fargate

AWS Fargate is a serverless compute engine specifically for containers. It eliminates the need to manage EC2 instances or clusters.

Benefits:
  • No server management required
  • Pay only for resources used
  • Automatic scaling
  • Works with both ECS and EKS

AWS Lambda

AWS Lambda is a serverless compute service that runs code in response to events without provisioning or managing servers.

Features:
  • Event-driven execution
  • Automatic scaling
  • Pay per request
  • No server management

AWS Elastic Beanstalk

AWS Elastic Beanstalk is a Platform as a Service (PaaS) for application deployment. Developers upload their code, and AWS automatically manages the infrastructure.

Managed Aspects:
  • Scaling
  • Monitoring
  • Load balancing
  • Application health monitoring

Amazon EBS (Elastic Block Store)

Amazon EBS provides persistent block storage volumes for use with EC2 instances. EBS volumes are highly available and reliable storage volumes that can be attached to any running instance in the same Availability Zone.

Features:
  • Persistent storage independent of instance lifecycle
  • High availability and durability
  • Multiple volume types for different use cases
  • Automatic backups with snapshots

Service Comparison

Different compute services offer varying levels of control and management responsibility.

Control vs. Management:
  • EC2: Offers the most control but requires the most management effort
  • Containers (ECS, EKS, Fargate): Improve portability and efficiency
  • Lambda: Provides the lowest operational overhead with event-driven execution
  • Elastic Beanstalk: Balances ease of use with infrastructure control
