Unit 7 – Storage Services

Amazon EBS • Amazon S3 • Amazon EFS • S3 Glacier

Unit 7 Overview

AWS Unit 7 focuses on cloud storage in AWS, explaining how data is stored, accessed, protected, and managed across different AWS storage services. Storage is a critical component of cloud computing because applications rely on fast, reliable, and scalable data access.

This unit explores the four main storage types used in AWS: block storage, object storage, file storage, and archival storage. Each storage type serves a different purpose depending on performance needs, scalability requirements, cost considerations, and how often the data is accessed. By the end of this unit, you should be able to choose the most appropriate storage service for a given workload.

Storage Basics

Storage in AWS is designed to be scalable, durable, and highly available, meaning it can grow as data increases, protect against data loss, and remain accessible even during failures. AWS provides multiple storage services because no single storage solution fits every application.

Different applications require different access speeds, sharing capabilities, and cost models. For example, operating systems and databases need low-latency storage, while backups and archives prioritize durability and low cost. Understanding these differences allows organizations to store data efficiently while minimizing cost and maximizing performance.

Amazon EBS (Elastic Block Store)

Amazon EBS is a block storage service designed specifically for use with Amazon EC2 instances. It functions like a virtual hard drive that can be attached to an instance and used to store operating systems, applications, and databases.

EBS provides persistent storage, meaning data remains available even if the EC2 instance is stopped or restarted. This makes EBS ideal for workloads that require consistent performance and reliable data persistence.

EBS – How It Works

EBS volumes are created within a specific Availability Zone and can only be attached to EC2 instances in the same zone. Each volume acts as raw block-level storage, allowing the operating system to format and manage it just like a physical disk.

Volumes can be detached from one instance and reattached to another, which is useful for recovery or maintenance. AWS also supports EBS snapshots, which create point-in-time backups stored in Amazon S3 for durability and disaster recovery.

EBS – Volume Types

Amazon EBS offers multiple volume types optimized for different workloads. SSD-backed volumes are designed for frequent access, low latency, and high IOPS, making them suitable for databases and application servers.

HDD-backed volumes are optimized for large, sequential workloads such as log processing or data warehousing. Choosing the correct volume type helps balance performance and cost, ensuring applications run efficiently without overpaying for unused performance.

Amazon S3 (Simple Storage Service)

Amazon S3 is an object storage service designed to store and retrieve virtually unlimited amounts of data. Instead of storing data as files or blocks, S3 stores data as objects inside containers called buckets.

S3 is built for extreme durability (99.999999999%), global accessibility, and scalability. It is commonly used for backups, static website hosting, media storage, and large-scale data analytics.

S3 – Objects & Buckets

Each S3 object consists of the data itself, metadata, and a unique key that identifies the object within a bucket. Buckets must have globally unique names and define where data is stored.

S3 supports features such as versioning, which keeps multiple versions of an object, and encryption, which protects data at rest. These features make S3 both flexible and secure for storing critical data.

S3 – Storage Classes

Amazon S3 provides multiple storage classes to reduce cost based on access frequency. S3 Standard is used for frequently accessed data, while Standard-IA and One Zone-IA are designed for infrequently accessed data at a lower cost.

S3 Intelligent-Tiering automatically moves data between tiers based on usage patterns, and S3 Glacier is used for long-term archival storage. Storage classes allow organizations to optimize cost without sacrificing durability.

S3 – Use Cases

Amazon S3 is commonly used for:

  • Application data storage
  • Backups and disaster recovery
  • Static website hosting
  • Media storage and streaming
  • Data lakes and analytics

Because S3 is highly scalable and accessible over the internet, it is often the backbone of cloud-based applications.

Amazon EFS (Elastic File System)

Amazon EFS is a file storage service that provides a shared file system for multiple EC2 instances. Unlike EBS, which attaches to one instance at a time, EFS allows simultaneous access from many instances.

EFS automatically scales as files are added or removed, eliminating the need for capacity planning. This makes it ideal for workloads that require shared access to files.

EFS – How It Works

EFS uses standard file system protocols, allowing applications to access files just like they would on a traditional file server. Storage capacity grows and shrinks automatically based on usage.

Because EFS spans multiple Availability Zones, it provides high availability and fault tolerance without manual configuration.

EFS – Benefits

EFS is well suited for collaborative workloads such as:

  • Content management systems
  • Web servers with shared content
  • Development and testing environments
  • Machine learning workflows

Its ability to support shared access simplifies application architecture and improves scalability.

Amazon S3 Glacier – Use Cases

Amazon S3 Glacier is a low-cost archival storage service designed for data that is rarely accessed but must be retained for long periods. Retrieval times range from minutes to hours depending on the retrieval option selected.

Glacier is commonly used for compliance records, long-term backups, and historical archives. While access is slower, the significantly lower storage cost makes it ideal for long-term retention.

Storage Comparison

  • EBS: Block storage for EC2 with low latency and high performance
  • S3: Object storage that is highly scalable, durable, and internet accessible
  • EFS: File storage that supports shared access across multiple instances
  • S3 Glacier: Archival storage optimized for low cost and infrequent access

Choosing the right storage service depends on performance needs, access patterns, and budget.
