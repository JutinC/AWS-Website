Unit 1

Benefits of Cloud Computing Cost-effective – no upfront hardware costs Scalable – scale up or down instantly Reliable – built-in redundancy Secure – AWS manages infrastructure security Flexible – wide range of services for different use cases Cloud Computing Deployment Models Cloud-based – everything runs in the cloud On-premises – everything runs locally Hybrid – combination of cloud and on-prem Cloud Computing Service Models IaaS (Infrastructure as a Service) You manage OS, apps, and data AWS manages hardware Example: Amazon EC2 PaaS (Platform as a Service) AWS manages OS and infrastructure You manage apps and data Example: Elastic Beanstalk SaaS (Software as a Service) Fully managed software Example: Gmail, Salesforce Key Vocabulary Cloud Computing Pay-as-you-go Scalability Elasticity High Availability Fault Tolerance IaaS / PaaS / SaaS Hybrid Cloud On-demand resources

Unit 2

UNIT 2: AWS Global Infrastructure AWS Global Structure Region – A geographic area (e.g., us-east-1) Availability Zone (AZ) – One or more data centers within a region Edge Location – Used for content delivery (CloudFront) Best practice: Deploy across multiple AZs for high availability. Why AWS Uses Regions and AZs Low latency Disaster recovery Compliance with data laws High availability AWS Shared Responsibility Model AWS and the customer share security responsibilities. AWS is responsible for: Physical data centers Hardware Networking infrastructure Customer is responsible for: Data Applications OS updates IAM permissions Mnemonic: AWS secures the cloud, YOU secure what’s in the cloud. Key Vocabulary Region Availability Zone (AZ) Edge Location High Availability Fault Tolerance Shared Responsibility Model Disaster Recovery Latency

Unit 3

Compute Services Amazon EC2 (Elastic Compute Cloud) Virtual servers in the cloud You choose: Instance type (CPU, RAM) OS Storage Highly customizable and scalable EC2 Instance Types General Purpose – balanced performance Compute Optimized – CPU-heavy tasks Memory Optimized – RAM-heavy tasks Storage Optimized – high I/O Storage Services Amazon S3 (Simple Storage Service) Object storage (files, images, videos) Scales automatically Highly durable (11 9’s durability) Uses buckets and objects Amazon EBS (Elastic Block Store) Block storage for EC2 Acts like a hard drive Must be attached to an EC2 instance Amazon EFS (Elastic File System) Shared file storage Multiple EC2 instances can access at once Networking Services Amazon VPC (Virtual Private Cloud) Your private network in AWS You control IP ranges, subnets, routing Security Groups Virtual firewalls Control inbound and outbound traffic Applied at the instance level Subnets A range of IP addresses in a VPC Can be public or private Key Vocabulary EC2 Instance AMI (Amazon Machine Image) S3 Bucket Object Storage EBS EFS VPC Subnet Security Group Inbound / Outbound Rules
